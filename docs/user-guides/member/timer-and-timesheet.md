# Member: Timer and timesheet

## Start a timer

1. Go to **Timer** (`/timer`).
2. Choose a **project** and **task** (only projects you are on appear).
3. Click **Start**.
4. The timer runs until you click **Stop**.
5. Stopping creates a time entry with source “timer” on your timesheet.

One active timer per user at a time.

## View your week (calendar)

1. Go to **Timesheet** (`/timesheet`).
2. Switch **day**, **week**, or **month** view.
3. Drag on the grid to create entries; drag blocks to move; resize edges.
4. Toggle **occupied slots** to see busy time from other workspaces.

## Time tracker (list view)

1. Go to **Time tracker** (`/time-tracker`).
2. Browse entries grouped by week with pagination.
3. Edit or delete entries inline (respects timesheet lock when submitted).

## Submit for approval

If your project requires approval, see the full guide: **[Timesheet submissions and approval](../timesheet-submissions-and-approval.md)**.

Short version:

1. Go to **Submissions** (`/submissions`).
2. Use **Action needed** for periods you still need to submit or fix.
3. Add an optional note → **Submit** for that period only.

Locked entries cannot be edited until approved, rejected, or an edit request is granted.

## Add a manual entry

1. On **Timesheet**, use **Add entry** (or equivalent control).
2. Select task, start and end date/time, optional description.
3. Save — the entry appears with source “manual”.

Manual entries can be edited or deleted. Timer-created entries should be treated as read-only in the UI (adjust time via manual entry if your process allows).

## Tasks

1. Go to **Tasks** (`/tasks`) to see tasks for your projects.
2. Tasks are created by admins on each project.

## Tips

- Pick the correct task before starting the timer to avoid reallocation later.
- Use billable flags consistently if your workspace tracks billable vs non-billable work.
