# Testing

> **QA / manual testers:** see the plain-language [QA testing guide](../user-guides/qa/testing-guide.md) (setup, checklists, sign-off). This page is the technical reference for developers.

## Commands

From the repository root:

```bash
pnpm test              # unit tests (all packages, via Turborepo)
pnpm test:coverage     # coverage: api module floors + contracts gate + ui
pnpm test:integration  # API Supertest e2e (real Postgres + Redis)
pnpm test:e2e          # API integration + admin Playwright + client Playwright
pnpm test:dashboard    # local test hub (coverage HTML, Playwright reports, Swagger link)
pnpm test:ui           # Vitest interactive UI (API unit tests)
```

### Test hub (GUI)

`pnpm test:dashboard` opens **http://localhost:9321** — a local hub (similar in spirit to Swagger for the API) that links to:

| Artifact                           | Generated by                                        |
| ---------------------------------- | --------------------------------------------------- |
| API / contracts / UI coverage HTML | `pnpm test:coverage`                                |
| Admin / client Playwright HTML     | `pnpm --filter @chronomint/{admin,client} test:e2e` |
| Swagger UI                         | `pnpm --filter @chronomint/api dev` → `/api/docs`   |

Interactive runners (separate terminals):

- `pnpm test:ui` — Vitest UI for API units
- `pnpm --filter @chronomint/admin test:e2e:ui` — Playwright UI (admin)
- `pnpm --filter @chronomint/client test:e2e:ui` — Playwright UI (client)

Package-specific:

```bash
pnpm --filter @chronomint/admin test:e2e
pnpm --filter @chronomint/client test:e2e
pnpm --filter @chronomint/ui test
```

## Test pyramid

| Layer           | Runner             | Location                                                       | CI job        |
| --------------- | ------------------ | -------------------------------------------------------------- | ------------- |
| Unit            | Vitest             | `apps/api`, `packages/contracts`, `packages/ui`, `apps/client` | `unit`        |
| API integration | Vitest + Supertest | `apps/api/test/**/*.e2e.ts`                                    | `integration` |
| Browser e2e     | Playwright         | `apps/admin/e2e`, `apps/client/e2e`                            | `e2e`         |

CI workflow (`.github/workflows/ci.yml`) runs four stages:

1. **quality** — format, lint, typecheck, build (Turborepo + optional remote cache)
2. **unit** — `pnpm test:coverage` + coverage/JUnit artifacts
3. **integration** — migrate, seed, `pnpm test:integration` + JUnit artifact
4. **e2e** — migrate, seed, API + admin dev servers, admin + client Playwright + HTML/JUnit artifacts

Artifact retention: 7 days.

## Coverage gates

Baseline snapshot: [`coverage-baseline.json`](./coverage-baseline.json).

| Package              | Enforcement                                                               |
| -------------------- | ------------------------------------------------------------------------- |
| `packages/contracts` | Vitest thresholds: ≥90% lines/statements, ≥70% branches, ≥25% functions   |
| `apps/api`           | Per-module **application** floors via `scripts/check-module-coverage.mjs` |
| `packages/ui`        | Informational only (artifact uploaded)                                    |

API module floors and targets live in [`apps/api/scripts/module-floors.json`](../../apps/api/scripts/module-floors.json). Floors are set below current coverage and ramp toward **70%** per module over 2–3 sprints. Global API gate (30%+) deferred until global lines ≥35%.

```bash
pnpm test:coverage
```

Outputs:

| Package              | LCOV                                    | Summary                                             |
| -------------------- | --------------------------------------- | --------------------------------------------------- |
| `apps/api`           | `apps/api/coverage/lcov.info`           | `apps/api/coverage/coverage-summary.json`           |
| `packages/contracts` | `packages/contracts/coverage/lcov.info` | `packages/contracts/coverage/coverage-summary.json` |
| `packages/ui`        | `packages/ui/coverage/lcov.info`        | `packages/ui/coverage/coverage-summary.json`        |

JUnit unit reports:

## Pre-commit test gate

Husky runs `scripts/check-staged-has-tests.mjs` after lint-staged. Staged production code must include matching tests:

| Change                             | Required staged test                           |
| ---------------------------------- | ---------------------------------------------- |
| API `application/` or `interface/` | Module `*.spec.ts` or `apps/api/test/*.e2e.ts` |
| `packages/contracts` source        | `*.spec.ts`                                    |
| `packages/ui` component            | sibling `*.spec.tsx`                           |
| Admin/client `features/`           | `e2e/` spec or unit `*.spec.ts`                |

Emergency bypass: `SKIP_TEST_CHECK=1 git commit ...`

Cursor agents: use skill `.cursor/skills/chronomint-test-delivery/SKILL.md` and rule `.cursor/rules/testing-tdd.mdc`.

## Admin browser e2e

- Config: `apps/admin/playwright.config.ts`
- Helper: `apps/admin/e2e/helpers/auth.ts` (`loginAsAdmin`)
- Specs: `smoke`, `categories`, `projects`

```

```

## Turborepo

Root scripts `build`, `lint`, `typecheck`, and `test` use Turborepo (`turbo.json`). Optional remote cache in CI:

1. Create a Vercel account / team or self-hosted remote cache
2. In GitHub repo **Settings → Secrets and variables → Actions**:
   - Secret: `TURBO_TOKEN` (token from Vercel → Remote Cache)
   - Variable: `TURBO_TEAM` (team slug, e.g. `team_xxxx`)
3. CI jobs `quality` and `unit` already pass these env vars when set; without them, local `.turbo` cache still works

Local cache directory: `.turbo`

- `e2e` — admin + client Playwright

Run `pnpm test:coverage` locally before opening a PR.
